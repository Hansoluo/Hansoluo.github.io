SSD 论文笔记

May 16, 2018

理解SSD

  • 巧妙利用了多重特征图，加快了速度
  • 一次运行，利用anchors或者prior/default boxes直接完成目标分类和BBox回归
  • 按照SSD300的模型，在不同的卷积层上生成一共7308个prior boxes,针对每个box区域都预测出class置信度和BBox的位置，再进行阈值过滤和NMS处理

论文
ssd-caffe
ssd-keras

Abstract

  • SSD的方法是在特征图上取一系列不同宽高比和尺寸的默认框（事先定义好）去匹配真实的边界框
  • 多重特征图的预测可以处理不同大小的图片
  • SSD与区域推荐算法完全不同的是一点是，无需区域生成或者再采样，目标检测算法所有的过程都在单个的神经网络中实现

Introduction

  • 区域推荐算法速度慢，不适合实时目标检测
  • SSD的做法：
    • 用一个小的卷积核预测目标的类别和边界框位置的补偿
    • 用另外的卷积核为不同比例的检测器进行预测，并且对不同尺寸的特征图上进行
  • SSD的核心是在不同的特征图上用卷积核针对一系列固定大小的默认框去进行目标分类和位置预测

The Single Shot Detector (SSD)

模型

  • 多尺度特征图检测
  • 卷积预测层
    • VGG16作为基础网络
    • 每一个新加的层都用卷积去预测出结果
  • 默认框(default boxes)和比例
    • 将默认框和对应的特征图的单元格进行关联
    • 对于一个特征图单元格，预测目标类别和针对默认框形状作微调（offset）
    • 类似于Faster R-CNN的 anchor boxes

训练

  • 匹配策略

    • 需要计算 ground truth boxes 和 default boxes的 IOU 值
    • 一个 ground truth box 对应 多个 default boxes
    • 选择 IOU阀值 大于 0.5 的进行匹配

  • 训练目标

    • 假设一个 ground truth box 匹配上 N个 default boxes，它的损失为 default boxes的损失函数除以个数 N ，相当于取平均值
    • 对一个default box，损失函数为 定位损失 和 置信度损失 之和
    • 定位损失采取 L1 平滑方法，针对边界框的中心点坐标和宽度高度进行回归
    • 分类采用 softmax 损失函数

  • 为 default boxes 选择 尺寸 和 长宽比

    • 特征图的重要作用，不仅尺寸越来越小，而且提炼的都是有用信息，特征具有不变性
    • 低层和高级网络的特征图都可以有效提供图像语义切割的成绩
    • SSD同时利用低级和高级特征图进行检测
    • 不同特征图具有不同感受范围，defaultboxes的大小不用特意去设计，而是根据特征图去相应的生成相应的大小

  • 负面数据过多

    • 在匹配的过程中，大量错误的default boxes生成，远远超过正确的数量
    • 为避免数据不平衡导致的训练不对等，取置信度高的样本
    • 控制正负比例 为 3：1

  • 数据增强

    • 使用所有的原图
    • 采用IOU比 0.1/0.3.0.5.0.7.0.9 来批量取样
    • 随机采样，采样图像为原始图像大小的[0.1，1]，长宽比在1/2和2之间
    • 光度变形
    • 水平翻转

Experimental Results

  • PASCAL VOC2007

    • SSD300模型: 使用conv4_3，conv7（fc7），conv8_2，conv9_2，conv10_2和conv11_2来预测位置和置信度
    • 对于conv4_3，conv10_2和conv11_2，只关联了4个默认边界框。对于所有其它层，放置了6个默认边界框
    • SSD300比Faster R-CNN好1.1%，SSD512比Faster R-CNN好3.6%
    • SSD对边界框大小很敏感。它在较小目标上检测性能要差，增加输入尺寸（例如从300×300到512×512）可以帮助改进检测小目标

  • 模型分析

    • 数据增强很重要
    • 更多的默认边界框比例会更好
    • 多个不同分辨率的输出层更好

  • VOC2007测试后结论

SSD300是唯一可以取得70%以上mAP的实现检测方法。通过使用更大的输入图像，SSD512在精度上超过了所有方法同时保持近似实时的速度。

Related Work

  • 与RCNN方法比较

    • 最初的R-CNN方法需要对成千上万的裁剪图像进行分类，这是昂贵和耗时的
    • SPPnet加快了原有的R-CNN方法，引入了空间金字塔，并使用多个分辨率下的特征图
    • Faster R-CNN用RPN取代了选择性搜索，进行ROI提取，通过交替两个网络之间的微调共享卷积层和预测层将RPN和Fast R-CNN结合在一起
    • SSD与Faster R-CNN中的区域提出网络（RPN）非常相似，因为我们也使用一组固定的（默认）边界框(prior/default boxes)进行预测，类似于RPN中的锚边界框（anchor boxes）
    • 但是SSD不需要利用Roi-pool后的特征进行分类，而是在每个目标类别在每个边界框中同时生成一个分数，相比叫来说更容易训练和更快

  • 与YOLO/OverFeat比较

    • OverFeat是滑动窗口方法的深度版本，在获得底层目标类别的置信度之后，直接预测边界框
    • YOLO直接对图像的最顶层的特征图进行处理，预测类别和边界框置信度
    • 如果不使用多个特征图进行多个默认框的预测，而是直接基于最顶层的特征图上使用默认框，则和overfeat类似
    • 如果直接在最顶层的特征图加上一个全连接层进行预测来代替卷积预测器，则近似YOLO

Conclusions

  • 本文介绍了SSD，一种快速的单次多类别目标检测器。我们模型的一个关键特性是使用网络顶部多个特征映射的多尺度卷积边界框输出
  • 大量仔细选择默认边界框会提高性能，SSD模型比现有方法至少要多一个数量级的边界框预测采样位置，尺度和长宽比
  • SSD在准确性和速度方面与其对应的最先进的目标检测器相比毫不逊色
