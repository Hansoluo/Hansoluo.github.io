pytorch和keras的比较

发表于 2018-05-11
字数统计: 字

一句话的感受是：

如果说 tensorflow 则是 C的话，keras是简化版的C，而pytorch 则是 python 。

最主要的两个区别：

  • 层级
  • 计算图

比如说都是对 tensor 进行操作，tensorflow缺乏好用的数据类型,而pytorch却有可以实现自动求导的Variable，就像python中非常好用的list数据类型。

在实际使用中的区别：

数据预处理和加载

在数据的加载方便，这一度是困扰很多人的脏活，keras和pytorch都有可以用来加载数据集的工具，但是用法有有点不同。

keras 中的数据处理

  • 常规的数据预处理流程：
    • 定义变量X和Y，明确shape
    • 读取和转化图片数据，需用到opencv
    • 遍历图片，存入 X 和 Y
    • 使用sklearn.model_selection的train_test_split完成训练和测试数据集的切分
  • 数据增强：
    • 通过ImageDataGenerator来实现对图片的各种增强操作
    • 通过datagen.flow方法实例化一个针对图像batch的生成器，用于给模型的输入
  • label 数据理是手动处理的，和pytorch较大的区别

pytorch 中的数据处理

在pytorch 分为两步中有数据集和数据加载。

  • Dataset.ImageFolder(root, transform, target_transform, loader)
    • 读取文件夹root –> loader加载图片 –> transform转换 –> target_transform
    • transform提供了对图片数据的各种增强、归一转化等操作
    • label是按图片数据不同类的文件夹名顺序来自动编码的，即{类名：类序号}，一般从0开始，比如 cat:0 dog:1，如果不清楚，可以通过self.class_to_idx属性来查看
    • 实际负责对原始数据的处理(Image –> input_data + label)
  • Dataloader(dataset, batch_size=1, shuffle=False, sampler=None, num_workers=0, collate_fn=default_collate, pin_memory=False, drop_last=False)
    • dataset: 前一步构建好的数据集
    • batch_size: CNN的训练一般都是分批，每次对一个batch的数据进行操作，比全部喂入SGD的准确率和速度更好。需要注意batch越大，占用内存越大
    • shuffle: 随机打乱 num_workers：多进程加载数
    • 数据以何种方式喂入训练模型，单条数据实际上还是[image,label]
    • dataloader是一个可迭代的对象

构建计算图

最主要的差别在于keras和tensorflow都是基于静态图的方法来计算的，而pytorch是动态的方式来生成的。举官方的两个例子来对比，最能说明问题。

tensorflow的方式

  • 使用图(graph)来表示计算任务
  • 在会话(session)中的执行图

keras使用的是静态图的方式，涉及优化器的选择、损失函数的定义，模型的保存等等。
训练模型时，把事先定义好的X_tarin和Y_train，喂入模型直接训练
model.fit(X_train, y_train, batch_size=32, epochs=16......)

torch的方式

Pytorch则需要动态地构建可自动求导的变量Variable,喂入模型，定义loss进行反向传播求导。
loss.backward()

结论

kears更简单点，但是也可能过封装了。Pytorch的动态构建需要遵循tensor–>Variable–>model的层级逐步定义完成，比keras麻烦一点，但是极大的利用了python的语法，增加了灵活性。

使用keras，更多的是调用函数，而几乎不用写过多的代码；使用pytorch，则天然的用python语言，来灵活构建所需代码。
