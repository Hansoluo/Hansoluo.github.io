pytorch入门笔记

May 9, 2018

深度学习火热的年代，各种框架工具层出不穷。最近pytorch和tensorflow有一统江湖的感觉,关于两者之间的差别可以看下之间的一篇文章

为什么选择pytorch

  • 常用深度学习框架对比
    • Theano：已经停止更新
    • Tensorflow: Goggle强推的工具，较底层，功能强大，最流行
    • Keras： 针对Tensorflow等框架的层层封装，易用，不够灵活
    • Caffe2: CV领域利器，性能优越，文档少，推广少
    • MXNet: 文档混乱，但分布式性能强大，适合AWS云平台
    • CNTK： 语音领域利器，文档少，推广少

  • github指标

    • pytorch的 github 数据： 6,774 repositorys
    • keras 的 github 数据：10,809 repositorys

  • Pytorch的优点

    • python first
    • 简洁（代码少）
    • 速度（释放内存，运行快）
    • 易用（API接口简单易懂）
    • 流行（文档完善，社区活跃）

  • fast.ai 放弃 keras+tensorflow 选择 Pytorch

    • 用keras无法实现 attention 模型，tensorflow实现太过复杂，而pytorch可以
    • tensorflow的静态图，则好比黑箱操作，无法了解内部。pytorch不是基于静态计算图，而是动态框架，可以像python一样交互式的运行和debug，方便调试
    • Introducing Pytorch for fast.ai

pytorch的基本组成

pytorch的整个层级是按照 tensor –> varaiable(autograd) –> nn.Module 从高到低组成的。基本的tesor组成可以自动求导的variable数据类型

  • tensor
    • 张量，可以看做是多维数组
      • 文本，一维数组
      • 音频，二维数组
      • 彩色图片，三维数组（第三维为颜色通道）
    • 运算基本同numpy，具有良好的继承性
    • 维度变形操作用 view 函数，区别于numpy中的 reshape
    • 可通过.cuda方法转化为GPU的Tensor，实现加速运算
  • Variable
    • 对data、grad_fn、grad的封装
      • data和grad是数据，grad是data对应的梯度
      • grad_fn是method，用来反向传播时计算输入的梯度
      • 类似于python的list数据类型，自动的将数据排列好，并赋予一个下标
    • 调用Variable的backward()，会进行反向传播自动微分计算梯度
    • grad在反向传播过程从事累加的，意味着每次进行反向传播，梯度都会累加之前的梯度，所以每次进行反向传播之前需要把梯度清零
  • nn
    • 主要是nn.Module模块，用于搭建各种深度神经网络模型

pytorch的数据加载

  • 手动加载，使用np.loadtxt或pandas读取，定义x和y的Variable。在建立forward pass时，传递给model。这种方法适用于少量数据，无法使用batch。
  • Dataset 数据集
    • __init__: 读取数据，并定义x和y
    • __getitem__: 返回一条数据(x,y)
    • __len__: 返回数据集的长度
  • Dataloader 数据加载
    • dataset: 前一步构建好的数据集
    • batch_size: CNN的训练一般都是分批，每次对一个batch的数据进行操作，比全部喂入SGD的准确率和速度更好。需要注意batch越大，占用内存越大
    • shuffle: 随机打乱 num_workers：多进程加载数
    • num_workers: 进程数，可以并行加载，提高速度

应用基本流程

  • Dataset
  • Dataloader
  • Design your model using class
    • all by hand
    • select from PyTorch API
  • Construct loss and optimizer (select from Pytorch API)
  • Training cycle (forward, backward, update)
  • Validate
  • Test
