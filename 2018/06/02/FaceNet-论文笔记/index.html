Facenet

  • DCNN直接优化embedding
  • triplet loss

论文 被引用次数 1508
Github Code 1
Github Code 2

摘要

  • 将人脸图像映射到欧几里得空间，用空间距离衡量人脸的相似性
  • 得到人脸向量后，人脸识别、验证和聚类很容易实现
  • 每个人脸使用128维向量表征

引言

  • 同一个人有较近的距离，不同的人有较远的距离
  • 之前人脸识别算法直接使用DCNN作为分类器，中间瓶颈层作为一个人脸的表征。如果模型要具备好的泛化能力，则瓶颈层表征维数非常大，1000或者更多（应该是指FC层）
  • triplet包含两张匹配的人脸，和一张非匹配的人脸，目的是用一对正例，和一对负例，并保证ap和an之间差一个margin
  • 训练样本使用的是切割紧凑的人脸图片，不需要2D或3D对齐
  • 提出在线负样本挖掘策略，确保在网络训练过程中持续增加triplets的难度
  • 为提高聚类正确性，使用hard正样本挖掘技术，来确保单个人脸的EMbedding约束在球形内


（图1：姿态和光照不变性——Facenet可以改善姿态和光照问题。0表示没有距离，4是距离最大，设置阈值为1.1可以进行较好的人脸识别）

相关工作

  • 基于ZF和inception网络进行构建
  • 深度学习+PCA+SVM方法
  • 用siamese网络优化L1距离
  • 用25个小型网络融合，每个网络对应于人脸的不同区块，使用分类和验证损失函数

方法

  • 使用embedding，将图像映射到特征空间R（d维）
  • 三元损失可以加强在正样本对和负样本对之间的margin，使得同一类的人脸存在多种情况，同时因为加强了margin，因此可以区别其他的人脸


（图2：模型架构——DCNN+L2正则化+嵌入向量+三元损失）
（图3：三元损失尽量缩小锚例和正例的距离，增大锚例和负例的距离）

  • 三元损失
    • 锚示例是样本集中随机选取的一个样本
    • 正示例与锚示例属于同一个人的脸
    • 负示例与锚示例属于不同人的脸

  • 三元组选择

    • 计算所有可能的triplet容易满足损失公式，这些对于训练和结果收敛速度没有贡献
    • 为了快速收敛，选择违反公式的triplet具有决定性作用
    • 选择hard-positive和hard-nagetive
      • 每N步后线下寻找triplet，使用最近生成的网络模型，计算argmin和argmax在数据的一个子集上
      • 在线获取triplet，通过从一个小的batch中选取hard positive或者hard negative样本
    • 本文使用在线获取，使用大的mini-batches包含几千个样本，而且只计算在这个mini-batche上的argmin和argmax
    • 在训练样本中采样大约40个人脸图像每个人每个mini-batch。此外，随机采样负样本人脸，加入到每一个mini-batch
    • 选取mini-batch中所有的ap对而不是选择最难的ap对，选择最难的an对
      • 选取所有的ap对能在初始阶段增强收敛稳定性和速度

  • 深度卷积网络

    • 使用BP算法和SGD优化来训练CNN，AdaGrad用来调整步长
    • a设置为0.2
    • 基于两个基础网络
      • ZF网络
      • GoogLenet

  • 数据集

    • LFW数据集
    • YouTube Faces数据集

试验

  • 使用5种网络架构，对比模型大小和验证精度，inception 224x224精度最高
  • LFW数据集精度达到 98.87% ，超过当前最好成绩 99.63%
  • YouTube Faces数据集达到 95.12%，超过 DeepID2的成绩 93.2%

结论

  • 提出了将人脸映射到欧式空间做度量学习的方法用于人脸识别
  • 模型需要对齐后的人脸数据
  • 后续针对模型大小和训练时间做改进
