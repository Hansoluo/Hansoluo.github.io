dl102_ch4_NNLM-深度学习笔记1-神经网络语言模型

发表于 2017-11-14

重点

  • 理解神经网络的矩阵表示形式
  • 了解神经网络语言模型的优势
  • 理解 Word embedding 的概念（中文可解释为 “词嵌入”，Word embedding 作用是将源数据映射到另一个空间）

视频1：更深入的理解神经网络

  • 深度学习算法本质还是梯度下降
  • 以一个最简单的神经网络来推导w和b
  • 链式法则 -> 剥洋葱 -> 反向传播算法
  • 矩阵乘法

视频2：神经网络的表达能力

  • 表达能力指的能够表达什么样的x到y的映射关系（函数）
  • 理论可证明单隐层有限的节点数可以逼近任意形状的函数，但实际应用不可取，节点数庞大
  • 隐层越多，结果跟准确，运算更快
  • 以如何学画画举例（类似于认知学中的组块理论）

视频3：矩阵符合表示神经网络

  • W * X -> 输入数据X表达为列向量
  • 多样本的矩阵乘法
  • X * W -> 输入数据 X 表达为行向量
  • tf.layers.dense 类似于 矩阵乘法

视频4：神经网络语言模型

  • 统计语言模型存在的问题
    • N-grama 无法判断两个很像的词 如今天 和 明天
    • 本质是只靠查字典方式，无法推理和迁移
  • 神经网络语言模型
    • 用于情感分类的二分类问题是输出一个概率，表示正面情绪的强度即可
    • 语言模型要素：
      • 输出一个概率分布，有N个词，就有N个输出
      • 加和等于1
    • sigmoid -> softmax 归一化指数函数
      • 对数据进行归一化处理，所有数据和为1
      • 权重保持不变，凸显大的值
    • 输入：上下文
    • 词向量(word embedding)->权重的向量
    • 稀疏矩阵
  • 为什么NN_LM可以迁移知识？
    • 所有样本进行全局优化
    • 今天和明天在相似的上下文里面，词向量类似
    • 今天有而明天没有的一些上下文，明天可以GET到

  • 单隐层无线逼近理论
  • 词向量
  • Softmax function

Hexo博客搭建

发表于 2017-10-07

搭建博客

  • 搭建过程
    • 第一次浅尝辄止，看了教程觉得很麻烦，而且不理解，就只是看了下
    • 第二次是看人家的教程，有两个问题不清楚，中途放弃（其实答案很简单）
      • 是否需要购买域名？（不一定需要）
      • 不需要域名如何设置GitHub_page？ (在仓库的Settings选项里勾选)
    • 第三次，用google搜索了几篇，写得很简洁，看明白之后，于是动手布置成功
  • 参考文档
    • How to use Hexo and deploy to GitHub Pages
    • Hexo 博客搭建指南
    • 使用 Hexo 为自己在 Github 上建一个静态 Blog 站点
    • 史上最详细的Hexo博客搭建图文教程

原理

  • hexo 是一个轻量级的基于Node.js的静态博客框架
    • 里面已经将各种写博客需要的框架都规定好了
  • theme 类似于atom的主题插件，可以随意选择安装更换
    • 在这里我使用的是Next主题，在网站的底部会有显示
    • 参照Next官方文档，可以完成很多页面的布局和设置，比如添加侧边栏、标签和搜索等功能

部署

  • hexo本身只能完成本地的部署，不可以完成公网的部署
  • 公网部署的原理是利用github生成的GitHub_page，生成独立域名后，再利用它来进行公网部署，这样别人也可以来看你的博客

文章编写及发布流程

  • 流程：
    • 使用草稿模式hexo new draft "文章标题"写草稿
    • 用hexo new "文章标题"，会在source->_posts文件夹内生成一个”文章标题”.md文件，而且和里面的Front-matter中的title对应起来
    • 本地文章markdown预览(atom编辑器)
    • 本地服务器预览命令 hexo s -debug
    • 发布命令 hexo cgd（缩写）

  • 使用以下命令进行发布:

    hexo clean # 删除本地静态文件（Public目录）

    hexo g # 生成本地静态文件（Public目录）

    hexo d # 将本地静态文件推送至github

  • 后面两个命令可以简化为一个hexo g -d

  • 发布文章后，如果对文章修改后重新发布，则无需hexo new，其他同发布流程
