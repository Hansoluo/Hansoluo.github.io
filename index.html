搭建博客过程

  • 这是我的博客，在这里随便写点什么东西进行测试
  • hexo博客搭建完成后写的第一篇
  • 其实之前有搭建过两次，但是都没有成功
    • 第一次浅尝辄止，看了教程觉得很麻烦，而且不理解，就只是看了下
    • 第二次是看人家的教程，有的说需要购买域名，有的说不需要购买域名，搞糊涂了。不需要购买域名的说需要github_page，但是不知道如何设置，也没有去搜索，就放弃了
    • 第三次，看了李笑来的教程，他写得很简单，我并没有按照他的来做，因为我觉得对里面的过程还是不清楚。但是，他提到遇到问题一定要用google，而且记住要看英文文档而不是中文文档
    • 搜了几篇英文文档，写得很简洁，但是我看明白了，于是动手布置成功

原理

  • hexo 是一个轻量级的博客的模板
    • 里面已经将各种写博客需要的框架都规定好了
  • theme 类似于atom的主题插件，可以随意选择安装更换
    • 在这里我使用的是next主题，在网站的底部会有显示

部署

  • hexo 本身只能完成本地的部署，不可以完成公网的部署
  • 部署的原理似乎是 利用github生成的gitpage，有独立的域名，可以利用它来进行公网部署，这样别人也可以来看了

文章编写及发布流程

  • 流程：
    • 用hexo new "文章标题"，会在source->_posts文件夹内生成一个”文章标题”.md文件，而且和里面的Front-matter中的title对应起来
    • 本地文章markdown预览
    • 本地服务器预览 hexo cgs（缩写）
    • 发布 hexo cgd（缩写）

  • 使用以下命令进行发布:

    hexo clean # 删除本地静态文件（Public目录）

    hexo g # 生成本地静态文件（Public目录）

    hexo d # 将本地静态文件推送至github

  • 发布文章后，如果对文章修改后重新发布，则无需hexo new，其他同发布流程
